# Lark Lite 项目说明（改进版）

## 目标与背景
本项目研究飞书多维表格开放接口，并将其封装为基于 Web 标准 `fetch` 的 HTTP 读写实现。  
这样可降低因官方接口变化频繁带来的研发成本与维护风险。

## 设计原则
- TypeScript 实现，类型清晰、可维护
- 底层只依赖 Web 标准 `fetch`，避免引入第三方依赖
- 同时支持浏览器、Bun、Node.js 三种运行环境
- 架构可拆分、可拷贝、可复用

## 架构约束
采用分层可拆分架构：
- **core 兼容层**：最小可用的 fetch 收发实现，确保三端一致
- **功能层**：构建在 core 之上，提供具体 API / 工具函数
- **使用方式**：用户可直接拷贝项目根目录、`core` 及所需功能文件夹使用

## 目录与配置约定
- `package.json`：项目配置
  - 目标零依赖（如必须引入依赖需给出明确理由）
  - 如可行，提供针对不同运行环境的更精简打包脚本
- `src/`：源码根目录
- `src/core/`：核心兼容层实现
- `src/{xxx}/`：功能模块目录（`{xxx}` 可变）
